Uso de la etiqueta INPUT y sus atributos

Etiqueta INPUT

Es una de las etiquetas mas importantes para la solicitud de datos mediante un formulario, y que permite espedificar los tipos de datos que se quiere solicutar para enviar.

Tambien establece las reglas de validacion de los datos.

Atributo accept

este atributo se usa especificamente para los datos de tipo file, y permite especificar los tipos de archivos que acepta el servidor que resibe los datos.

Posibles valores del atributo accept
Valor del Atributo Descripción
Audio/* Todos los archivos de audio son aceptados.
Video/* Todos los archivos de video son aceptados.
image/* Todos los archivos de imagenes son aceptados.
MIME tipe Sea de especificar un valor de tipo MIME, sin parametros.

Ejemplo con el atributo accept

Atributo alt

Permite espesificar un texto alternativo para la imagen. Este texto alternativo es visualizado cuando se produce un error en el atributo src, la conexión fuese muy lenta o si el usuario utiliza un lector de pantalla.

Se puede esepecificar siempre y cuando el tipo de dato que se solicito, sea de tipo image, según se haya especificado en el atributo type.

Ejemplo con el atributo ALT



Atributo autocomplete

Especifica que el dato solcitado puede utilizar las entradas de datos anteriores que pueda haber realizado el usuario, para autocompletar el campo que se esta solicitando.

Se pueden establecer como valor posible on, que indica la activación. Y off, indicando que el auto completado estaria desactivado

Atributo autofocus

Permite especificar cuál es el control que resibe el "foco", cuando se realiza la carga del documento web en el navegador. Es un atributo booleano y solo con aparecer significa que esta activo.

Se a de tener en cuenta que este atributo solo se puede especificar en un solo dato del formulario.

Atributo checked

El atributo se puede utilizar en el caso de que en el atributo type, se haya especificado un tipo de dato checkbox, o radio. u permite especificar que el campo sobre el cual se aplica parezca inicial mente marcado, cuando se realiza la carga del docuemto.

Atributo disabled

El atributo especificado que el elemento sobre el que se aplica, se ha de desactivar. Un elemento desativado está bloqueado y no se puede cambiar ni realizar clic sobre el mismo.

Este atributo es de tipo booleano es de sir con tansolo que se especifique en la etiqueta ya estaria activi

De esta forma se puede especificar un campo incluso fuera del formulario, pero indicando que pertenece un formulario en concreto, de manera que al enviar los datos se envien tambien el contenido de campo esterno al formulario,pero relacionado.

Atributo formaction

Permite especificar una URL distinta de envio, sobre escribienso la que aparece en el atributo acction, del formulario. De esta forma es posible situar distintos botones de envio de formulario, enviando los datos a diferentes sitios segun se pulse un boton u otro.

El atrubuto se a de usar cuado el atributo type tenga un valor submit o image

Atributo formmethod

Permite especificar la forma o metodo del envido datos contenidos en el formulario.

Los valores para el atributo son GET o POST. Sobre escribiendo el metodo definido en el formulario, segun el boton sobre cual se pulse, se acepta el atributo en boton de envio de type submt o image.

Las reglas de envió dependiendo del valor del atributo get o post, son las mismas comentadas en el atriburo method de la etiqueta form.

Ejemplo con el atributo fromenctype y formmethod



Atributo formnovalidate

Permite especificar que los datos del formulario no sean validados, segun las carateristicas de los campos utilizadosen el formulario, de esta forma el formulario envía los datos sin realizar nungún tipo de comprobacion de los datos.

Este atributo se usa sobre los campos de tipo submit o image, y se super pone al atributo apliicado al formulario, permitiendo asi el envio de datos no validados.




Atributo formtarget

El atributo permite definir el destino por defecto para todos los enlaces y formularios del documento web. El atributo se a de aplicar en campos de tipo submit o image, y sobre escribe el destino aplicado al formulario.

Atributo height.

Permite definir el alto de una imagen en pixeles. el atributo solo se aplica a campos de tipo image, y se suele usar para los botones de envió de formulario

Atributo width.

Permite especificar el ancho de la imagene ne pixeles. El atributo se aplica solo en campos de tipo image, y se suele usar para los botones de envió de formulario

Atributo name.

Permite asignar el nombre que se el quiere definir al campo del formulario. El noombre puede ser usado bien por un lenguaje de cliente, para interactuar con el formulari, bien por lenguaje de servidor para procesar los datos recibidos.

Ejemplos con los atributos width, heigtht y name



Atributo readonly.

Permite especificar que un campo sea de solo lectura, es decir que su contenido podra ser seleccionable y visible pero no midificable. Se suele con conjuntos de lengujes de programacion de cliente, para alterar el valor de solo lectura, cuando se cumpla cierta condicion en los datos.

Este atributo es booleano es decir que con tan solo que aparesca en la etiqueta ya estaria activo.
