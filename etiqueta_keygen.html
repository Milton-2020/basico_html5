uso de la etiqueta keygen y sus atributos

Etiqueta keygen

Permite crear una clave codificada, de forma que sean enviadas junto con los datos del formulario, para añadir un nivel más de seguridad en el envío y recepción de los mismos.

Cuando el formulario es enviado la clave privada es almacenada en el ordenador cliente, mientras que la clave pública, es enviada al servidor.

Atributos.

autocus: el atributo permite especificar cuál es el control que recibe el "foco", cuando se realiza la carga del documento weben el navegador. Es un atributo booleano y solo con aparecer significa que está activado.

disabled: el atributo especificar que el elemento sobre el que se a aplicado, se ha de desactivar. Un elemento desactivado está bloqueado y no se puede cambiar ni realizar clic sobre el mismo.

form: el atributo especificar a qué formulario pertenece el campo que se astá solicitando espesificando el atributo identificador del formulaio. Como valor se ha de especificar el valor del atributo id, asociado al formulario donde se encuentra incluido la etiqueta.

name: el atributo permite especificar el nombre que se quiere asignar al campo del formulario. el nombre puede ser utilizado posteriormete bien por lenguajes cliente, para interactuar con el formulario, bien con leguaje de servidor para procesar los datos recibidos.

challege: El atributo permite especificar que las claves generadas sean cambiadas automáticamente, despúes de haberse producido el envió de los datos.

keytype: El atributo permite especificar el tipo de codificacion que se quiere utilizar para el cifrado de las claves. Se puede seleccionar tres tipos de codificacion: rsa,dsa, y ec. Siendo la utilizada por defecto el primer tipo de codificación.
