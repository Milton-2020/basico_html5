Uso de la etiqueta FORM y sus atributos

Etiqueta FORM

Diha etiquera permite definir un formulario de datos, utilizado para traspasar datos al servidor.

Un formulario puede contener a su vez, elementos de entrada como lo son campos de texto, casillas de verificación, botones radio, botones de envió, lista de selección, areas de texto, etc.

Atributos de la etiqueta Form

Atributo accept-charset: este atributo permite establecer el conjunto de caracteres que a de utilizar el formulario para envio de informacion al servidor.

Sino se establece el atributo se toma por defecto el valor "UNKNOWN" ("Desconocido"), el cual aplica el conjunto de caracteres que tenaga aplicado el documento web. (los atributos que mas se utilizan son UTF-8 y ISO-8859-1 que corresponde al alfabeto latín)

Atributo action: el atributo permite especificar donde han de ser enviados los datos del formulario, cuando este es enviado. Se a de especificar la tuta relativa o absoluta al recurso encargado de procesar la información.

Atributo autocomplete: el atributo especifica que el formulario puede utilizar las entradas de datos anteriores que puede haber realizado el usuario, para autocompletar los campos que son solicitados por el formulario. Los valores que puede tener son on u off, en on esta activo el auto completado en off esta desavlitado

Atributo enctype: el atributo permite especificar como han de ser codificado los datos de un formulario, cuando este es enviado. Solo se puede usar cuando el atributo method, tiene asignado un valor "POST"

Atributo method: el atributo permite especificar la forma o método de envió de los datos contenidos en el formulario.

Los valores para el atributo son GET, o POST.

El primer valor GET envia los datos utilizando la URL, por lo tanto los datos son enviados mediante nombre/valor, no se recomienda usar este valor cuando los datos son sencibles a la seguridad, ya que los valores son mostrados en la barra de direcciones del navegador.La cantidas de informacion maxima que se puede enviar es de 3000 caracteres.

El segundo valor del atrbuto es POST por el contrario no tiene limite de tamaño y los datos se envian dentro de la peticion HTTP, dek atributo action por lo tanto los datos no son visible en si envio y la seguridad de los mismos de mantiene intacta.

Atributo name: el atributo pemite especificar el nombre que se quiere asignar al formulario o los datos que se solicitane en el mismo. El nombre puede ser utilizado posteriormente bien por lenguajes cliente, para interactuar con el formulario, bien con lenguajes de servidor para procesar los datos recibidos.

Atributo novalidate: el atributo permite especificar que los datos del formulario no sean validados, agún lasd características de los campos utilizadas en el formulario de esta forma el formulario envía los datos sin realizar ningún tipo de comprobación sobre los datos.

Este atributo booleano, por lo tanto su aparición en la etiqueta de formulario, hace que se active el mismo.

Atributo target: el atributo permite especificar el destino por defecto para todos los enlaces y formularios del documento web.
